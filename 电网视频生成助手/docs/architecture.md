# 架构草案

## 一、目标

构建一个能自动把电网资讯转成短视频生产包的智能体，既能吃 RPA 数据，也能直接联网抓官方电网新闻。

## 二、推荐链路

```text
信息源
  -> RPA采集 / 互联网抓取
  -> 清洗去重
  -> 网页正文抽取
  -> 来源可靠性打分
  -> 重要性评分
  -> 视频选题生成
  -> 脚本/分镜/标题生成
  -> 字幕与预览渲染
  -> 配音与自动剪辑
  -> 发布到抖音/视频号
```

## 三、建议模块

### 1. 采集代理

负责从网站、公众号、行业平台抓取文本和链接，输出标准 JSON。

### 2. 审核代理

负责去重、判断来源可靠性、过滤敏感和无法核实的信息。

### 3. 选题代理

负责判断哪些资讯值得做成视频，并提炼核心看点。

### 4. 脚本代理

负责生成：

- 标题
- 封面文案
- 60 秒口播
- 分镜建议
- 字幕稿
- 标签

### 5. 制作代理

负责把脚本分发给：

- GIF/分镜预览
- TTS
- 数字人口播
- 自动剪辑
- 发布机器人

## 四、生产建议

### 视频类型

- 快讯型：30 到 60 秒，适合政策、负荷、事故、市场规则
- 讲解型：60 到 90 秒，适合热点机制拆解
- 盘点型：90 秒左右，适合一周电网要闻

### 合规重点

- 不发布未证实事故信息
- 不夸大电网风险
- 涉及具体数据时标注来源
- 不把预测说成事实
- 政策与知识内容要保留原始出处，便于复核

## 五、下一阶段开发建议

1. 接 RPA 输出 JSON
2. 扩展重点来源的专用解析器
3. 对接真实大模型
4. 增加素材模板库和账号风格模板
5. 打通配音和自动剪辑
6. 增加发布审批流
